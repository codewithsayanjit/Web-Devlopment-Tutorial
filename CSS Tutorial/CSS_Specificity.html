CSS Specificity

This document provides an overview of CSS specificity rules.

Specificity Rules

CSS specificity determines which styles are applied to an element when multiple rules match it:

  1. Important Selector( highest specificity)
    Syntax:   !important

    Example:   color: red !important;
    The Specificity of !important is higher than any other specificity.
    The Specificity of Important Selector is considered as 10000.

  2. Inline styles (higher specificity)

    The Specificity of Inline Style is 1000.

  3. ID selectors

    The Specificity of ID Selector is 100.

  4. Class selectors, attribute selectors, and pseudo-classes

    The Specificity of Class Selector is 10.

  5. Element selectors and pseudo-elements (lowest specificity)

    The Specificity of Element Selector is 1.

  6. Universal selector (*) has no specificity

    The Specificity of Universal Selector is 0.

Specificity Calculation

To calculate the specificity of a selector, sum the values based on the rules above. The specificity is represented as a four-part value (a,b,c,d), where:

  • a = number of inline styles
  • b = number of ID selectors
  • c = number of class selectors, attribute selectors, and pseudo-classes
  • d = number of element selectors and pseudo-elements

For example, the selector div#header .nav li:hover has a specificity of (1+100+10+10+1) = 122:

  • 1 ID selector (#header) = 100
  • 1 class selector (.nav) = 10
  • 1 pseudo-class (:hover) = 10
  • 2 element selectors (div, li) = 1 + 1 = 2

Thus, the total specificity is 100 + 10 + 10 + 2 = 122.
